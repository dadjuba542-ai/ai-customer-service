# 案例档案 v1 验收与发布准备清单

更新时间：2026-06-30

## 验收目标

把“案例档案推荐、统一案例抽屉、后台链接识别填充”收口成一个可稳定使用的 v1 模块。当前阶段不新增前台案例栏目，不替代公司已有案例小程序。

## 自动化验证

当前已通过：

```bash
node --check static/app.js
python3 -m py_compile services/case_recognition_service.py routes/cases.py
python3 scripts/test_case_documents.py
python3 scripts/test_today_features.py
```

覆盖范围：

- 案例表和 FTS 表初始化。
- 5 条虚拟案例 seed。
- 便秘、腹胀、益生菌等症状/产品匹配。
- 隐藏案例不参与推荐。
- `/api/chat/send` 返回 `related_cases`。
- 案例公开详情、标签筛选列表。
- 后台案例新增、编辑、隐藏、删除。
- 链接识别接口鉴权。
- 非 HTTP、本地、内网链接拒绝。
- 模拟 H5 页面识别。
- AI 结构化抽取成功路径。
- AI 失败时规则降级。
- 识别预览不直接入库，确认保存后可搜索。
- 今日功能回归：团队配置、团队统计、社区问答、评论审核、点赞等。

## 浏览器手动验收清单

启动服务：

```bash
python3 app.py
```

后台入口必须使用：

```text
http://127.0.0.1:5001/admin
```

不要直接打开：

```text
file:///Users/test/Downloads/ai-customer-service/templates/admin.html
```

直接打开模板文件会导致登录接口 `fetch` 失败。

### 后台案例管理

- 登录后台成功。
- 进入“案例档案”。
- 新增一条测试案例，保存成功。
- 编辑刚新增的案例，保存成功。
- 隐藏该案例后，前台推荐不再出现。
- 重新显示该案例后，前台可再次推荐。
- 删除测试案例，列表中不再出现。

### 链接识别填充

- 在“从链接识别案例”输入公开 H5/网页链接。
- 点击“识别案例”后出现可编辑预览。
- 预览中包含标题、客户画像、症状标签、产品标签、场景、摘要、正文；来源链接只用于识别说明，不作为单条案例跳转。
- 点击“填入当前表单”后，案例表单字段被正确回填。
- 点击“确认创建案例”后，案例被保存到列表。
- 使用无法访问或非网页链接时，页面提示失败，且不会直接写入数据库。

### 前台推荐与抽屉

- 前台提问：`我妈便秘腹胀适合什么`。
- 机器人回答下方出现“相关案例”卡片。
- 只有存在更多本次问题相关案例时，才显示“查看更多”。
- 点击“查看更多”后，只展示本次问题相关案例，不进入全部案例列表。
- 点击案例卡片打开统一案例抽屉详情。
- 在详情里点击症状标签，当前抽屉直接切换为该标签相关案例列表。
- 点击“查看相似案例”，当前抽屉直接切换为相似案例列表。
- 从列表点击其他案例，当前抽屉切回详情。
- 从列表进入详情时，返回按钮能回到上一次列表。
- 不再出现案例列表弹层和案例详情弹层上下叠加。
- 后台配置“客户案例库 H5 链接”后，案例详情底部显示“查看更多客户案例”。
- 清空“客户案例库 H5 链接”后，刷新前台，案例详情不显示“查看更多客户案例”。
- 首页、底部导航、发现页不出现案例库入口。

### 移动端 UI

- 在手机宽度下，聊天回答内容不超出边框。
- 相关案例卡片不超出边框。
- 案例抽屉内容可滚动，关闭按钮可见。
- 标签、相似案例、返回按钮都可点击。

## 发布前检查

- 生产环境设置 `SECRET_KEY`。
- Railway 持久化时设置 `DATABASE_DIR=/data`。
- 配置 `COZE_API_KEY` 或在后台系统配置里保存 Coze API Key。
- 确认 `static/uploads/` 在部署环境可写且会被持久化或有替代对象存储方案。
- 确认真实案例链接为公开 H5/网页；小程序私有路径只能保留外链，需要人工补正文。

## 当前结论

- 自动化验证：已通过。
- 本地运行态接口：已启动 Flask 服务，`GET /admin`、`GET /api/cases` 可响应，`POST /api/admin/cases/recognize-link` 未登录时正确返回鉴权错误。
- 浏览器后台操作：需要管理员登录后继续人工点验新增、编辑、隐藏、删除、链接识别和前台抽屉交互。
- 外部案例库入口：需要人工填写一个真实 H5 链接后确认案例详情按钮跳转。
- 发布建议：先用 3-5 条真实客户案例链接完成手动验收，再考虑上线或继续做 CSV 批量导入。
